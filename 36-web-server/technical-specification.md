# Техническое задание — Веб-сервер на Express в BAS

> ТЗ, на базе которого создавался модуль. Автоматизация браузера с использованием **BAS**: взаимодействие двух серверов и бота, эмуляция человеческого поведения при заполнении формы, обработка кода верификации, загрузка изображения и логирование.

[← К модулю](README.md)

## 1. Общий процесс

Автоматизация включает взаимодействие между двумя серверами и ботом на основе BAS. Цель — эмуляция человеческого поведения при заполнении формы на сайте, обработка кода верификации, загрузка изображения и логирование выполнения задачи.

## 2. Последовательность действий

### Этап 1. Начало работы

1. **Сервер №1** отправляет на **Сервер №2 (BAS)** HTTP-запрос с:
   - `id` пользователя
   - имя
   - телефон
2. Сервер №2 (BAS) принимает данные, запускает бота и открывает сайт:
   - `https://your-site.example/form/<form-id>` _(подставьте свой URL формы)_

### Этап 2. Эмуляция активности пользователя

Бот имитирует человеческое поведение:
- прокручивает страницу вверх/вниз;
- двигает мышью случайным образом;
- ждёт случайное время перед действиями.

### Этап 3. Заполнение формы

1. Находит поле **Имя** по селектору и вводит данные.
2. Находит поле **Телефон** по селектору и вводит данные.
3. Находит поле для **верификационного кода**.

### Этап 4. Запрос кода верификации

1. Бот отправляет HTTP-запрос на Сервер №1, передавая:
   - `id` пользователя;
   - дополнительные данные в заголовках/теле запроса (для предотвращения путаницы при параллельном запуске процессов).
2. Сервер №1 отправляет код верификации Серверу №2.
3. Сервер №2 (BAS) получает код и вводит его в форму.

### Этап 5. Загрузка изображения

1. Бот отправляет запрос на Сервер №1 для получения ссылки на изображение.
2. Сервер №1 отправляет Серверу №2 ссылку.
3. Сервер №2 (BAS) загружает изображение в поле формы.

### Этап 6. Завершение процесса

1. После успешной отправки формы загружается страница «Спасибо».
2. Бот сохраняет страницу «Спасибо» (например, на Google Drive) и получает ссылку.
3. Бот отправляет Серверу №1 ссылку на сохранённую страницу.
4. Бот формирует лог выполнения задачи и отправляет его Серверу №1.

## 3. Технические требования

**1. Многопрофильность и многобраузерность**
- разные браузерные профили;
- разные прокси и куки.

**2. Связь между процессами**
- каждому процессу назначается `id` пользователя;
- дополнительные заголовки или данные в теле запроса предотвращают путаницу между процессами.

**3. Имитация поведения пользователя**
- ввод данных и взаимодействие со страницей с задержками;
- прокрутка и перемещение мыши — случайное поведение.

**4. Логирование**
- дата и время выполнения;
- `id` пользователя и номер телефона;
- статус выполнения каждого этапа;
- ошибки (если возникли);
- отправка лога на Сервер №1.

**5. Видео-инструкция**
- пояснение настройки и работы BAS-бота;
- демонстрация процесса в реальном времени.

## 4. Итог

Полностью автоматизированный процесс: взаимодействие серверов, имитация реального пользователя, корректная обработка данных и логирование. Видео-инструкция по настройке и управлению BAS-ботом.

## Соответствие ТЗ и функций проекта

Как этапы ТЗ реализованы в [`BasWebServer.xml`](BasWebServer.xml):

| Этап ТЗ | Функция BAS |
|---|---|
| Приём запроса от Сервера №1, запуск бота | `LaunchServer` (`POST /task`, Bearer-токен) |
| Профиль, прокси, отпечаток | `LoadProfile` |
| Прогрев куками | `CookiesWarming` |
| Эмуляция активности (этап 2) | `EmulateActivity` |
| Открытие сайта и заполнение формы (этап 3) | `LoadFormAndFill` |
| Запрос кода верификации (этап 4) | `GetCode` |
| Получение и загрузка изображения (этап 5) | `GetImageLink` / `DownloadImage` |
| Сохранение страницы на Google Drive (этап 6) | `GetAccessToken` → `UploadScreenToDisk` |
| Отправка ссылки и лога на Сервер №1 | `SendScreenAndLogSuccess` |
